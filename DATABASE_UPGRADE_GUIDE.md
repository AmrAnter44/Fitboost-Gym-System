# 🔄 دليل تحديث قاعدة البيانات القديمة

## 📌 نظرة عامة

هذا الدليل يشرح كيفية تحديث أي ملف داتابيز قديم ليتوافق مع النظام الحالي.

---

## 🎯 ما الذي يقوم به التحديث؟

عند رفع ملف داتابيز قديم، النظام يقوم تلقائياً بـ:

### ✅ الإضافات
- إضافة جميع الجداول الجديدة (مثل `ClassBooking`)
- إضافة الأعمدة الجديدة للجداول الموجودة
- إضافة الـ indexes الجديدة لتحسين الأداء
- إضافة الـ constraints الجديدة

### ⚠️ الحفاظ على البيانات
- **لن يتم حذف أي بيانات موجودة**
- جميع الجداول والأعمدة القديمة تبقى كما هي
- العلاقات الموجودة لا تتأثر

---

## 🖥️ طريقة الاستخدام من الواجهة

### الطريقة 1: من صفحة الإعدادات

1. اذهب إلى: **الإعدادات** → **إعدادات قاعدة البيانات**
2. في قسم "تحديث داتابيز قديمة":
   - اضغط على "اختر ملف الداتابيز"
   - حدد ملف `.db` القديم
3. اختر أحد الخيارين:

   **أ. تحديث وتنزيل** 📥
   - يحدث الملف ويرجعه لك للتنزيل
   - الداتابيز الحالية لا تتغير
   - مناسب للتجربة أو المعاينة

   **ب. تحديث واستبدال** 🚀
   - يحدث الملف ويستبدل به الداتابيز الحالية
   - يعمل نسخة احتياطية تلقائياً
   - مناسب للنقل النهائي

---

## 💻 طريقة الاستخدام من الـ CLI

### 1. باستخدام npm scripts

```bash
# نقل داتابيز قديمة وتطبيق التحديثات
npm run db:migrate /path/to/old/gym.db
```

هذا الأمر سيقوم بـ:
- نسخ الداتابيز القديمة
- تطبيق جميع التحديثات
- استبدال الداتابيز الحالية

### 2. باستخدام السكريبت المخصص

```bash
# إعداد شامل مع تحديثات
npm run db:setup

# أو بملف محدد
node scripts/setup-database.js --db-path=/path/to/old.db
```

---

## 🔍 ما الذي يحدث خلف الكواليس؟

```
1. رفع الملف ✅
   ↓
2. التحقق من صحة الملف (SQLite integrity check) ✅
   ↓
3. عد الجداول الموجودة 📊
   ↓
4. إنشاء نسخة مؤقتة من الملف 📋
   ↓
5. تطبيق Prisma DB Push ⚙️
   (يضيف الجداول والأعمدة الجديدة)
   ↓
6. فحص السلامة النهائي ✅
   ↓
7. إما تنزيل الملف أو استبدال الداتابيز الحالية 🎯
```

## 📋 أمثلة عملية

### مثال 1: تحديث داتابيز من نسخة قديمة

```bash
# لديك: gym-old.db من نسخة قديمة
# تريد: تحديثها للنسخة الحالية

# الطريقة 1: عبر الواجهة
# - اذهب للإعدادات → قاعدة البيانات
# - ارفع gym-old.db
# - اختر "تحديث وتنزيل"
# - ستحصل على: gym-upgraded-TIMESTAMP.db

# الطريقة 2: عبر CLI
npm run db:migrate ~/Desktop/gym-old.db
```

### مثال 2: تحديث داتابيز واستبدالها

```bash
# لديك: backup.db من نسخة احتياطية
# تريد: استخدامها في النظام الحالي

# عبر الواجهة:
# - اذهب للإعدادات → قاعدة البيانات
# - ارفع backup.db
# - اختر "تحديث واستبدال"
# - سيتم استبدال الداتابيز الحالية

# عبر CLI:
npm run db:migrate ~/backups/backup.db
```

### مثال 3: تحديث عدة ملفات

```bash
# تحديث كل ملفات .db في مجلد
for file in ~/old-databases/*.db; do
  echo "Processing: $file"
  npm run db:migrate "$file"
  sleep 2
done
```

## ⚠️ ملاحظات مهمة

### ✅ افعل:
- احتفظ بنسخة احتياطية من الملف الأصلي قبل التحديث
- جرّب "تحديث وتنزيل" أولاً للتأكد
- تأكد من سلامة البيانات بعد التحديث
- اقرأ رسائل النجاح/الفشل بعناية

### ❌ لا تفعل:
- لا ترفع ملفات تالفة أو غير صحيحة
- لا تستبدل الداتابيز الحالية بدون نسخة احتياطية
- لا تقاطع عملية التحديث
- لا ترفع ملفات ليست SQLite

## 🔧 التحقق من نجاح التحديث

### 1. من الواجهة:
```
الإعدادات → قاعدة البيانات → فحص سلامة الداتابيز
```

يجب أن ترى:
- ✅ السلامة: الداتابيز سليمة
- ✅ الاتصال: الاتصال ناجح
- ✅ عدد الجداول: XX جدول

### 2. من CLI:
```bash
# فحص السلامة
npm run db:validate

# عد الجداول
sqlite3 prisma/gym.db "SELECT COUNT(*) FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%';"

# عرض الجداول الجديدة
sqlite3 prisma/gym.db "SELECT name FROM sqlite_master WHERE type='table' ORDER BY name;"
```

## 🚨 حل المشاكل

### المشكلة 1: "الملف ليس قاعدة بيانات SQLite صحيحة"

**الحل:**
- تأكد أن الملف امتداده `.db`
- تأكد أنه ملف SQLite فعلاً
- جرب فتحه بـ SQLite Browser أولاً

### المشكلة 2: "الداتابيز المرفوعة تالفة"

**الحل:**
```bash
# فحص السلامة يدوياً
sqlite3 your-file.db "PRAGMA integrity_check;"

# إذا كان التقرير "ok" لكن لا يزال هناك خطأ، جرب:
sqlite3 your-file.db ".recover" | sqlite3 recovered.db
```

### المشكلة 3: "فشل تطبيق التحديثات"

**الحل:**
- تأكد من صحة ملف `schema.prisma`
- جرب تطبيق التحديثات يدوياً:
```bash
DATABASE_URL="file:./your-file.db" npx prisma db push
```
